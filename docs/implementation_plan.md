# 动物声学行为分析与接触系统实施方案

## 1. 数据集概况与策略

您当前拥有的数据集具有极高的**多样性**，非常有利于论文中的特征对比分析：

- **全功能组（用于物种+情绪双重分析）：**
  - **猴 (Monkey)：** `alarm` , `call`, `angry`, `happy`
  - **狗 (Dog)：** `angry` , `happy`, `sad`(3类)
  - **原鸡 (Gallus gallus)：** `alarm` , `call`, `song`(3类)
  - **鹿 (Deer)：** `alarm` , `mating_call`, `song`(3类)
- **单功能组（主要用于物种区分/特征展示）：**
  - **大象（Elephant）：**只有`angry`(无法训练表情模型，因为缺乏对比数据，但可用于物种识别)。
  - **海豚（Dolphin）/抹香鲸（Sperm Whale）：**无情绪标签（用于展示水下声学特征、去噪处理）。

## 4.详细实施步骤与逻辑

### **第一步：深度清洗与标准化（预处理）**

这是系统最关键的基础阶段。为了消除录音设备差异、噪声以及时长不一的环境问题，建立一套严格的**DSP清洗定时器**。

#### **1.基础信号标准化**

所有音频在进入处理流程前，必须强制转换为统一格式：

- **采样率（Sample Rate）：设定为22,050 Hz**。
- **通道 (Channels)：统一转换为单通道 (Mono)**。
- **深（Bit Depth）：** 16位PCM。

#### **2.定制化数字滤波设计（Digital Filtering）**

不同物种的生活环境和发声频率不同，必须设计**滤波器**来消除环境底噪（如风声、水流声）。建议使用**4阶巴特沃斯（Butterworth）滤波器**。

| **物种组别**      | **典型噪声环境** | **项目类型** | **频率参数建议**               | **设计理由**                                                 |
| ----------------- | ---------------- | ------------ | ------------------------------ | ------------------------------------------------------------ |
| **大象**          | 旷野风声、树叶声 | **低通**     | $f_c = 1000 \text{ Hz}$        | 大象包含大量次声波和低频隆声（<100Hz）。保留低频核心区，滤除鸟叫等高频杂音。 |
| **海豚 / 抹香鲸** | 水流声 (红噪音)  | **高通**     | $f_c = 150 \text{ Hz}$         | 水下录音通常有强烈的低频水流冲击声（<100Hz）。必须滤除以保留听觉生物信号。 |
| **原鸡/鹿/鸟类**  | 旷野/农场背景声  | **带通**     | $200\text{Hz} - 8000\text{Hz}$ | 滤除200Hz以下的风声/电流声，同时滤除8kHz以上的极高频白噪声。 |
| **猴/狗**         | 室内生活噪音     | **带通**     | $80\text{Hz} - 8000\text{Hz}$  | 家养动物环境较杂，保留人耳听觉核心范围即可。                 |

#### **3.消除与归一化**

- **静音消除（Silence Removal）：**使用`librosa.effects.trim`，阈值设为`top_db=30`。删除音频首尾的静音段。
- **能量归一化（Amplitude Normalization）：**将每个音频的幅值线性缩放到**[-1, 1]**区间。

#### **4.音频切割与填充策略（Segmentation & Padding）**

为了适应模型输入，并增加数据量（Data Augmentation）：

- **标准时长：**  **5.0秒**。
- **滑动窗口（Overlapping）：设置50%的重叠率**（即步长stride = 2.5s）。
- **填充：**对于短于5秒的音频，采用**“循环填充（Loop Padding）”**，即重复播放该声音直到填满5秒，保持特征密度。

#### **5. 数据集划分策略**

- **训练（70%）** /**验证（15%）**/**测试（15%）**。

------

### **第二步：DSP特征分析与可视化**

本步骤旨在从物理与数学层面解构声音。

#### **1.时频分析方法的选型**

- **方案：**最终选择**STFT (短时傅里叶变换)** 。
- **参数：**加汉宁窗（Hanning Window），窗长2048，跳步512。
- **理由：**动物声音非平稳性（Non-stationary），FFT丢失时间信息，STFT能响铃展示海豚“哨声”的调频曲线。

#### **2.生物听觉特征提取：MFCC及其导数**

- **静态特征（Static MFCC）：** 13维。
- **动态特征（Delta & Delta-Delta）：**一阶导数（速度）+二阶导数（高度）。
- **最终矩阵：** **39维（特征）x 216（时间步）**。情绪往往在隐藏声音的“变化率”中，Delta特征至关重要。

#### **3.统计特征**

- **光谱质心**与**过零率（ZCR）**分析，用于在GUI中展示高频组（海豚）与低频组（大象）的物理差异。

### **第三步：物种分类模型 (Task 1 & 2 - Species ClassifiMonkeyion)**

本步骤通过构建深度学习模型，实现对7类动物声音的准确分类。将MFCC特征视为二维图像，使用**2D CNN**进行端到端训练。

#### **0. 模型处理流程图（系统流程图）**

纯文本

```
[ 输入: 特征矩阵 .npy (39 x 216) ]
          ⬇
[ 阶段 1: 特征提取 (CNN Backbone) ]
|-- Conv2d(32) -> BN -> ReLU -> MaxPool (提取边缘/轮廓)
|-- Conv2d(64) -> BN -> ReLU -> MaxPool (提取纹理/图案)
|-- Conv2d(128) -> BN -> ReLU -> MaxPool (提取高级语义)
          ⬇
[ 阶段 2: 特征聚合 (Aggregation) ]
|-- AdaptiveAvgPool2d(1, 1)  --> 强制压缩为 (128 x 1 x 1)
|   (确保无论输入时长微小差异，输出尺寸恒定)
          ⬇
[ 阶段 3: 分类决策 (ClassifiMonkeyion) ]
|-- Flatten (展平)
|-- Dropout(0.5) (随机失活，防止过拟合)
|-- Linear(128 -> 7) (全连接层映射到7个物种)
          ⬇
[ 输出: 7类物种概率 (Softmax) ]
```

#### **1.网络架构设计（模型架构）**

相比LSTM，CNN对短声音（如狗叫、鸟叫）的瞬时纹理捕捉能力更强，且效率更高。

- **输入层：** `(Batch_Size, 1, 39, 216)`。
- **骨干网络：** 3组边界块，层层提取、纹理和构造。
- **自适应层：** `AdaptiveAvgPool2d((1, 1))`，强制统一特征维度。
- **分类头：** `Flatten` -> `Dropout(0.5)`-> `Linear(128 -> 7)`。

#### **2. 训练配置**

- **损失：** `CrossEntropyLoss`
- **优化器：** `Adam`（lr=0.001）
- **纪元数：** 50
- **策略：**提前停止（验证集损失连续5次不下降则停止）。

------

### **第四步：正向翻译——情绪理解与混合融合认知（任务3 - 混合情绪识别）**

本步骤旨在构建一个能够“听懂”动物情绪的高级感知模块。借助动物声学数据兼具**频域纹理（Timbre）与时域变换（Temporal Evolution）的双重特性，本方案放弃了传统的单一网络，转而采用“CNN + Transformer混合融合架构（Hybrid Fusion Architecture）”**。该架构通过“双流处理”机制，在小样本数据上实现了更高的鲁棒性。

#### **1.混合网络架构设计（Hybrid Network Architecture）**

系统采用双分支稀疏处理结构，又负责“看”语谱图的纹理（空间特征），又负责“听”特征序列的变化（时间特征），最后在决策层进行特征融合。

**模型处理流程图：**

纯文本

```
[ 输入: MFCC特征矩阵 (Batch, 1, 39, 216) ]
          │
          ├──────────────────────────┐
          ⬇                          ⬇
【分支 A: 空间特征流 (CNN)】    【分支 B: 时序特征流 (Transformer)】
[ 老师傅 - 冻结参数 ]            [ 新学生 - 动态学习 ]
          │                          │
[ SpeciesCNN Backbone ]        [ Input Projection ] (39 -> 128 dim)
  (Conv1 -> Conv2 -> Conv3)          │
          │                    [ Positional Encoding ] (注入位置信息)
[ AdaptiveAvgPool2d ]                │
          │                    [ Transformer Encoder ] 
[ Flatten ]                    (Nhead=4, Layers=2, Dropout=0.4)
          │                          │
[ Feature Vector A (128) ]     [ Global Avg Pooling ]
          │                          │
          │                    [ Feature Vector B (128) ]
          └──────────┬───────────────┘
                     ⬇
           【 特征融合层 (Fusion Head) 】
           [ ConMonkeyenate (A + B) -> 256 dim ]
                     ⬇
           [ Linear (256 -> 128) + ReLU ]
                     ⬇
           [ Dropout (0.5) ] (强力抗过拟合)
                     ⬇
           [ Output Layer (Linear -> Num_Classes) ]
```

- **分支A（CNN - 迁移学习）：** - 加载任务2训练好的`species_classifier_best.pth`权重。
  - **策略：**放置所有的图层层参数（`requires_grad=False`）。这部分网络作为稳定的提取器，捕捉动物声音的基础音色纹理。
- **分支B（Transformer - 捕捉）：**
  - **位置编码（Positional Encoding）：**赋予模型对声音顺序的采集能力。
  - **自注意力机制（Self-Attention）：**捕获长距离依赖（例如：狗的连续吠叫中，第一声和最后一声的关联）。
  - **高Dropout设置：**编码器层的采用`dropout=0.4`，显着增强对噪声的忍受度。
- **融合头（Fusion Head）：** -将静态纹理特征与动态特征特征拼接，并通过全连接层进行非线性映射，输出最终情绪分类概率。

#### **2.差异化训练配置中心（Differentiated Configuration Center）**

为了应对不同物种数据的异质性，代码中集成了`SPECIES_CONFIG`配置中心，端点地调整策略：

| **配置项**               | **狗（狗）**   | **猴子（猴子）** | **鹿（鹿）**   | **原鸡（Gallus）** | **通用策略说明**                                             |
| ------------------------ | -------------- | ---------------- | -------------- | ------------------ | ------------------------------------------------------------ |
| **噪声注入（噪音水平）** | `0.005`（低）  | `0.015`(中)      | `0.010`(中)    | `0.015`(中)        | 时动态向特征矩阵添加高斯训练噪声，防止模型死记硬背。狗的数据底噪增加，故注入量调低。 |
| **类别权重（手动权重）** | {0:2.5, 1:1.2} | {0:2.0, 1:2.0}   | {0:3.5, 1:1.0} | {0:2.5, 1:2.0}     | 针对**数据不平衡**（如“报警”样本缺少“歌曲”），手册赋予稀缺类别更高的损失惩罚权（2.0-3.5倍）。 |

#### **3. 训练超参数与优化策略（优化策略）**

- **优化器升级：** - 使用`Adam`优化器，学习率设置为**`1e-4`(0.0001)**。相比任务2，降低学习率以适应Transformer的需求需求。
  - **权重衰减(Weight Decay)：**设置为`1e-4`(L2正则化)，限制模型参数的大小，平滑损失平面，进一步抑制转录因子。
- **动态学习率调整（Scheduler）：** -采用`ReduceLROnPlateau`。监控验证集损失，若连续10个Epoch不下降，则将学习率减半（Factor=0.5）。
- **损失函数：**
  - `CrossEntropyLoss`若未配置手动权重，则启用`compute_class_weight='balanced'`自动计算类别平衡权重。
- **训练轮次：**增加至**100 Epochs**，因为强正则化（Dropout+Weight Decay）会收敛速度，需要更多轮次保证充分学习。

#### **4.语义映射与输出（Semantic Mapping）**

系统输出的不仅仅是数字标签，而是结合上下文的语义信息：

- **输入：**混合模型识别出`Species: Dog`+ `Emotion: Angry`。
- **映射逻辑：** - 查询内部语义表：`Dog_Angry`-> *Level 3 Threat / High Arousal*。
- **输出给LLM：** - “检测到犬类目标，情绪状态为‘愤怒’（置信度92%）。声学特征显示具备强烈的攻击性（高频爆发）。请生成应对建议。”

### **第五步：逆向翻译——智能决策与生成式交互（任务4&5）**

本步骤旨在实现“说出”动物语言。

#### **1. 系统架构：感知-认知-表达闭环**

- **感知层（DSP + CNN/Transformer）：**负责“听”。输入音频，输出`[Species: Dog, Emotion: Angry]`。
- **认知层（LLM Agent）：**负责“想”。引入大语言模型 DeepSeek作为交互大脑。
  - **输入：** “你是一位动物行为学家。现在前面有一只愤怒的狗……”
  - **输出：** “建议保持安静。**响应声音特征：低沉、缓慢、缓慢的呜咽声。**”
- **表达层（Audio Generation）：**负责“说”。
  - **方案：** RAG（搜索增强）+ Text-to-Audio API。

#### **2. DSP后处理（生物约束）**

无论生成音频，必须经过DSP管道：

1. **重采样：**回到22050Hz。
2. **频带限制：**施加带通激励（如狗80-8000Hz），从而生成模型产生的非自然α影。

#### **3. GUI架构**

- **Tab 1 信号分析：**实时波形/变化/箱线图 + LLM 专家解读。
- **Tab 2 交互终端：**聊天气泡形式，接收 Tab 1 结果，生成对话语音。
- **Tab 3 模型看板：** -**自适应图表：**根据窗口大小缩放。
  - **展示内容：**混合矩阵（验证混合模型精度）+训练曲线（验证权重衰减抗过重大效果）。
